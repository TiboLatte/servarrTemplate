## Stack Components
* **Networking:** Gluetun (Wireguard VPN)
* **Downloading:** qBittorrent, Prowlarr (Indexers), FlareSolverr (Cloudflare Bypass)
* **Management:** Radarr (Movies), Sonarr (TV Shows), Bazarr (Subtitles)
* **Streaming:** Jellyfin (Media Server), Jellyseerr (Requests)
* **Dashboard:** Homarr (Entry point), Dashdot (System Monitoring)

---

## Quick Start

### 1. Prepare the Files
Create a folder on your Linux server and place your `docker-compose.yml` and `.env` files inside.
```bash
mkdir media-stack && cd media-stack
```

### 2. Configure Environment Variables
Create your `.env` file based on the provided template.
```bash
nano .env
```
> **Important:** Ensure you fill in your `WG_PRIVATE_KEY` (from Mullvad) and your `USB_PATH` (where your hard drive is mounted).


### 3. Launch the Stack
Run the following command to pull images and start the containers:
```bash
docker compose up -d
```
## Port Mapping Reference

This table shows which ports you need to use in your browser to access the services. The format is **Host Port : Container Port**.

| Service | Host Port : Container Port | Access URL | Description |
| :--- | :--- | :--- | :--- |
| **qBittorrent** | `8080 : 8080` | `http://IP:8080` | Download Client Web UI |

## Service Integration Guide

Connecting the apps is the most important step for full automation.

### 1. Prowlarr & FlareSolverr (Cloudflare Bypass)
Some indexers are protected by Cloudflare. Prowlarr uses FlareSolverr to solve these challenges.
1.  In Prowlarr, go to **Settings > Indexers**.
2.  Add a **FlareSolverr** proxy. To do that click on the +, FlareSolverr, 
3.  **Host:** `http://localhost:8191` (Because both are inside the Gluetun network).
4.  **Tags:** Assign a tag like `flaresolverr`.
5.  **Usage:** On specific indexers requiring cloudflare bypass, add the `flaresolverr` tag so they route through the proxy.

### 2. Link Prowlarr to Radarr/Sonarr
1.  In Prowlarr, go to **Settings > Apps**.
2.  Add **Radarr** (and repeat for Sonarr).
3.  **Prowlarr Server:** `http://IP:9696` 
4.  **Radarr Server:** `http://IP:7878`.
5.  **API Key:** Copy the API Key from Radarr (and repeat for Sonarr) (**Settings > General**).
6. Add as many indexers as you want, you can filter by "public" in Privacy and in Categories. Test the connexion, if you're getting a cloudflare error, add tag "flaresolverr" at the  very end. 

### 3. Set up the Download Client (qBittorrent)
In Radarr and Sonarr:
1.  Go to **Settings > Download Clients**.
2.  Add **qBittorrent**.
3.  **Host:** `gluetun`.
4.  **Port:** `8080`.
5.  **Username/Password:** The ones you configured in the qBittorrent Web UI.

/!\ Usr and pwd are generated by qbitorrent the first time and should be available inside the container's logs.



## Path Mapping
To avoid "File not found" errors, all containers share a consistent root:
* **Configs:** `${USB_PATH}/docker_configs/`
* **Data:** `${USB_PATH}/data/` (containing `torrents`, `movies`, `tv`)
